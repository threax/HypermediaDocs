Threax Hypermedia Framework and Architecture

Updating to ExceptionFilter 6.0.0 and htmlrapier.halcyon 8.0.0

These two packages should be updated at the same time. The way the model state case is handled for errors was moved to the server side instead of the client side. No other action besides updating and rebuilding should be needed.

To do this add the following to your csproj:

<PackageReference Include="Threax.AspNetCore.ExceptionFilter" Version="6.0.0" />

And update package.json

"htmlrapier.halcyon": "^8.0.0",

This will ensure the libraries do the correct thing on both sides. Install packages and rebuild and all your errors should keep working, but the error state will be completly camel case instead of partially.
