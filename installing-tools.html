Installing Tools

You will need to install the following to get everything to work. This page shows using Visual Studio, but all you really need is the .Net Core SDK. With that you can use your preferred tools, you will just have to run some commands yourself.

Git

If you don't have git on your computer install Git for Windows and TortoiseGit. Use your real name and spc e-mail when prompted.

I also reccomend using the Windows Secure Channel Library when asked while installing git. Also choose Use Windows' default console window when asked.

Visual Studio

Install Visual Studio 2019 Community. Download

  1. When doing this you can select just .net core way at the bottom under Other Toolsets.
  2. Its a good idea to install the .net core 2.2 development tools. This may change if newer versions are available.
  3. Click Install and wait while Visual Studio is installed.
  4. When it is done installing open Visual Studio.

Client Side Development Tools

Next we will install the tools you need to make the client side stuff compile.

Make sure to run any commands below in a regular command prompt, not the Visual Studio Command Prompt.

  1. Install the NPM Task Runner plugin for Visual Studio. Download
  2. Also install Node.js latest reccomended version (10.15.0 lts at time of writing). Download
  3. Next install a couple of other node pacakages we use, which we install through npm
    1. Install typescript globally npm install -g typescript
    2. Install the threax-npm-tk globally as well npm install -g threax-npm-tk

Configure Visual Studio

Once these are installed you must configure Visual Studio.

  1. Search for External Web Tools in the Search Visual Studio box (Look in for it next to Help or press Ctrl + q).
  2. Make the order the following, if you have extras leave them at the end.
    1. .\node_modules\.bin
    2. $(PATH)
    3. $(VSInstalledExternalTools)
    4. $(VSINSTALLDIR)\Web\External
    5. $(VSINSTALLDIR)\Web\External\git

      You should have something similar to this screenshot when you are done.

Setup Additional Tools

The task runner explorer you installed above can be opened by typing Task Runner Explorer into the Quick Launch.

This can be used to run any of the client side build process scripts defined in your project.

You can also open the Test Explorer by typing Test Explorer into the quick launch.

This will show any tests in your project and allow you to run them.

Clone and Build the Template

The best project to test your environment with is the Threax.AspNetCore.Template project.

Clone the template to your machine with git from https://github.com/threax/Threax.AspNetCore.Template.git

Open AppTemplate.sln in Visual Studio.

Run the install task from Task Runner Explorer.

Click Build -> Rebuild Solution. This will probably take a while on the first try since it has to load a bunch of stuff from nuget.

Double Click build in the Task Runner Explorer to build the client side.

You should be able to run the app template and it should load the welcome page with correct css. If you try to do anything on this site you will get an error because the identity server is not setup, but the app itself should run. If you are prompted to trust the certificate do so. That will setup your machine to trust the dev certificate.

All of this should work without errors.

At this point all of the tools you need to build projects should be setup. The next step will be to get the identity server installed and configured.
