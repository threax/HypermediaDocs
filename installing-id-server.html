Installing Id Server

This page will describe how to install the Threax.IdServer Identity Server into your environment. With this you will be able to create user accounts and log into your apps. The reference identity server supports local users and application users, but you can customize it for your needs. This could include adding users authenticated through ADFS or external providers like Google or Microsoft. Any of the user login systems supported by .Net Core should be usable and you can build custom authentication if needed. The Identity Server will nicely unify all these logins and pass on the basic user information along with a guid. The guid is how each app will identify a user and discover their roles.

To setup the Id Server in your dev environment clone the Threax.IdServer repository from https://github.com/threax/Threax.IdServer.git.

Once it is cloned open the Threax.IdServer solution.

In Visual Studio run the install task in Task Runner Explorer for both the AppDashboard and the IdServer projects.

Run the build task in Task Runner Explorer for both of these projects as well.

Build the solution.

Change the run mode from IIS Express to SetupDb and run it.

Next change the run mode to Setup App Dashboard and run it.

Change back to IIS Express and run the id server and create an account. After logging in click on your user name in the navbar and click Manage.

From here make a note of the guid for your account.

Stop the program.

In the Package Manager Console in Visual Studio or on the command line run the following command from the Threax.IdServer folder:

dotnet run tools "addadmin {YourGuid}"

Replace {YourGuid} with the guid for your account, do not include the brackets.

This will setup the account you just created as an admin on the id server.

Next set the solution to run multiple projects and run both the IdServer and the AppDashboard. Log into the AppDashboard at https://localhost:44391.

If everything is working then your Identity Server and App Dashboard are setup correctly.
